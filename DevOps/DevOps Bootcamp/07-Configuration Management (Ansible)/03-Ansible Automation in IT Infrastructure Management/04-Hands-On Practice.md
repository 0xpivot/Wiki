## Hands-On Practice

To gain hands-on experience with Ansible, consider the following labs:

- **PortSwigger Web Security Academy**: Focuses on web application security but also includes sections on automation and scripting.
- **OWASP Juice Shop**: A deliberately insecure web application for practicing web security skills.
- **DVWA (Damn Vulnerable Web Application)**: Another intentionally vulnerable web application for learning web security.
- **WebGoat**: An interactive web application security training tool.

These labs provide a practical environment to apply the concepts learned in this chapter.

### Conclusion

In this chapter, we covered the fundamental concepts of variables, plays, and playbooks in Ansible. We explored how to define and use variables, the structure and usage of plays and playbooks, and how to orchestrate complex workflows. By following best practices and avoiding common pitfalls, you can create effective and maintainable Ansible playbooks.
