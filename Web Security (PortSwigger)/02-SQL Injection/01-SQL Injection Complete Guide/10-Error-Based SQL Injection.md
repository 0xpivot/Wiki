## Error-Based SQL Injection

### What is Error-Based SQL Injection?

Error-based SQL injection is a type of SQL injection attack where the attacker can extract information from the database by exploiting the error messages generated by the database server. This method relies on the fact that the application does not properly handle errors and instead displays them to the user, providing valuable insights into the underlying SQL query structure.

#### Why Does Error-Based SQL Injection Matter?

Error-based SQL injection is particularly dangerous because it allows attackers to gain detailed information about the database schema, including table names, column names, and even sensitive data such as usernames and passwords. By analyzing the error messages, attackers can craft precise SQL payloads to extract specific data.

#### How Does Error-Based SQL Injection Work?

When an application constructs SQL queries dynamically based on user input, it may inadvertently expose the internal structure of the query through error messages. For example, consider the following SQL query:

```sql
SELECT * FROM users WHERE username = 'admin' AND password = 'password';
```

If the application fails to sanitize user input properly, an attacker might inject a payload like `'; DROP TABLE users; --` into the username field, resulting in the following query:

```sql
SELECT * FROM users WHERE username = ''; DROP TABLE users; --' AND password = 'password';
```

This would cause the database to generate an error message indicating that the `DROP TABLE` statement is invalid, revealing the structure of the original query.

### Example of Error-Based SQL Injection

Consider a login form where the backend SQL query looks like this:

```sql
SELECT * FROM users WHERE username = '$username' AND password = '$password';
```

An attacker might input the following into the username field:

```plaintext
admin' OR '1'='1
```

This would modify the query to:

```sql
SELECT * FROM users WHERE username = 'admin' OR '1'='1' AND password = 'password';
```

The `'1'='1'` condition is always true, effectively bypassing the password check. If the application displays the error message, it might reveal details about the query structure, allowing the attacker to refine their payload further.

### How to Prevent / Defend Against Error-Based SQL Injection

#### Secure Coding Practices

1. **Parameterized Queries**: Use parameterized queries to ensure that user input is treated as data rather than executable code.
    ```python
    import sqlite3

    conn = sqlite3.connect('example.db')
    cursor = conn.cursor()

    username = 'admin'
    password = 'password'

    cursor.execute("SELECT * FROM users WHERE username = ? AND password = ?", (username, password))
    ```

2. **Stored Procedures**: Use stored procedures to encapsulate SQL logic and reduce the risk of SQL injection.

#### Detection and Monitoring

1. **Web Application Firewalls (WAF)**: Implement WAFs to detect and block suspicious SQL injection attempts.
2. **Logging and Monitoring**: Monitor application logs for unusual SQL queries and error messages that indicate potential SQL injection attacks.

### Union-Based SQL Injection

### What is Union-Based SQL Injection?

Union-based SQL injection is a technique that leverages the `UNION` operator to combine the results of two or more SELECT statements into a single result set. This method allows attackers to retrieve data from different tables within the same database.

#### Why Does Union-Based SQL Injection Matter?

Union-based SQL injection is significant because it enables attackers to access data from multiple tables, potentially exposing sensitive information such as usernames, passwords, and other critical data.

#### How Does Union-Based SQL Injection Work?

In a typical scenario, an application might construct a SQL query like this:

```sql
SELECT column1, column2 FROM table1 WHERE condition;
```

By injecting a `UNION` operator, an attacker can append another SELECT statement to retrieve data from a different table:

```sql
SELECT column1, column2 FROM table1 WHERE condition UNION SELECT column3, column4 FROM table2;
```

This combined query will return the results of both SELECT statements in a single result set.

### Example of Union-Based SQL Injection

Consider a search functionality where the backend SQL query looks like this:

```sql
SELECT title, description FROM articles WHERE id = $id;
```

An attacker might input the following into the ID field:

```plaintext
1 UNION SELECT username, password FROM users;
```

This would modify the query to:

```sql
SELECT title, description FROM articles WHERE id = 1 UNION SELECT username, password FROM users;
```

The result would include both the article data and the user credentials.

### How to Prevent / Defend Against Union-Based SQL Injection

#### Secure Coding Practices

1. **Input Validation**: Validate and sanitize user input to ensure it conforms to expected formats.
    ```python
    import re

    def validate_input(input_value):
        if re.match(r'^\d+$', input_value):
            return True
        return False

    id = '1 UNION SELECT username, password FROM users;'
    if validate_input(id):
        # Proceed with the query
        pass
    else:
        # Handle invalid input
        pass
    ```

2. **Least Privilege Principle**: Ensure that the database user account used by the application has the minimum necessary privileges to perform its tasks.

#### Detection and Monitoring

1. **Database Auditing**: Enable database auditing to track and log all SQL queries executed by the application.
2. **Real-Time Monitoring**: Use tools to monitor for suspicious SQL patterns in real-time.

### Inferential SQL Injection (Blind SQL Injection)

### What is Inferential SQL Injection?

Inferential SQL injection, also known as Blind SQL Injection, is a type of SQL injection attack where the attacker cannot directly view the results of the injected SQL query. Instead, the attacker must infer the correctness of their payload based on the behavior of the application.

#### Why Does Inferential SQL Injection Matter?

Blind SQL injection is particularly challenging because it requires the attacker to deduce the structure and contents of the database without direct feedback. This makes it harder to detect and defend against, as the attack is often subtle and persistent.

#### How Does Inferential SQL Injection Work?

There are two main types of blind SQL injection: Boolean-based and Time-based.

##### Boolean-Based SQL Injection

Boolean-based SQL injection involves crafting payloads that cause the application to respond differently based on the truth value of a condition. For example:

```sql
SELECT * FROM users WHERE username = 'admin' AND (SELECT CASE WHEN (1=1) THEN 1 ELSE 0 END);
```

If the condition `(1=1)` is true, the query will return results; otherwise, it will not.

##### Time-Based SQL Injection

Time-based SQL injection involves injecting payloads that cause the database to delay its response for a specified period of time. For example:

```sql
SELECT * FROM users WHERE username = 'admin' AND (SELECT CASE WHEN (1=1) THEN SLEEP(5) ELSE 0 END);
```

If the condition `(1=1)` is true, the database will wait for 5 seconds before responding.

### Example of Inferential SQL Injection

Consider a search functionality where the backend SQL query looks like this:

```sql
SELECT title, description FROM articles WHERE id = $id;
```

An attacker might input the following into the ID field:

```plaintext
1 AND IF(ASCII(SUBSTRING((SELECT username FROM users LIMIT 1),1,1))=97,SLEEP(5),0)
```

This would modify the query to:

```sql
SELECT title, description FROM articles WHERE id = 1 AND IF(ASCII(SUBSTRING((SELECT username FROM users LIMIT 1),1,1))=97,SLEEP(5),0);
```

If the first character of the first username in the `users` table is 'a', the database will wait for 5 seconds before responding.

### How to Prevent / Defend Against Inferential SQL Injection

#### Secure Coding Practices

1. **Input Validation**: Validate and sanitize user input to ensure it conforms to expected formats.
    ```python
    import re

    def validate_input(input_value):
        if re.match(r'^\d+$', input_value):
            return True
        return False

    id = '1 AND IF(ASCII(SUBSTRING((SELECT username FROM users LIMIT 1),1,1))=97,SLEEP(5),0)'
    if validate_input(id):
        # Proceed with the query
        pass
    else:
        # Handle invalid input
        pass
    ```

2. **Least Privilege Principle**: Ensure that the database user account used by the application has the minimum necessary privileges to perform its tasks.

#### Detection and Monitoring

1. **Database Auditing**: Enable database auditing to track and log all SQL queries executed by the application.
2. **Real-Time Monitoring**: Use tools to monitor for suspicious SQL patterns in real-time.

### Real-World Examples and Recent Breaches

#### Real-World Example: CVE-2021-22205

CVE-2021-22205 is a vulnerability in the WordPress plugin "WP Event Manager." An attacker could exploit this vulnerability to execute arbitrary SQL commands, leading to unauthorized access to sensitive data.

#### Real-World Example: Equifax Data Breach (2017)

The Equifax data breach in 2017 was caused by an unpatched Apache Struts vulnerability that allowed attackers to exploit a remote code execution flaw. This led to the exposure of sensitive personal data of millions of individuals.

### Conclusion

SQL injection remains one of the most prevalent and dangerous vulnerabilities in web applications. Understanding the various types of SQL injection—error-based, union-based, and inferential—is crucial for both attackers and defenders. By implementing robust security practices, validating user input, and monitoring application behavior, organizations can significantly reduce the risk of SQL injection attacks.
