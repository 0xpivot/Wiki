## Understanding SQL Injection

SQL Injection is a type of attack where an attacker manipulates a SQL query by inserting malicious input into a web application's form fields or URL parameters. This can lead to unauthorized data access, data manipulation, or even complete compromise of the database. SQL Injection vulnerabilities arise due to improper validation and sanitization of user inputs.

### How SQL Injection Works

When a web application constructs a SQL query using user-provided input without proper validation, an attacker can inject additional SQL commands to alter the intended behavior of the query. For instance, consider a simple login form where the username and password are submitted to the server:

```sql
SELECT * FROM users WHERE username = 'input_username' AND password = 'input_password';
```

If an attacker inputs `input_username` as `' OR '1'='1` and `input_password` as `' OR '1'='1`, the resulting SQL query becomes:

```sql
SELECT * FROM users WHERE username = '' OR '1'='1' AND password = '' OR '1'='1';
```

This query will return all rows from the `users` table because the condition `'1'='1'` is always true.

### Real-World Example: CVE-2021-27902

In 2021, a critical SQL Injection vulnerability was discovered in the popular WordPress plugin WPML (WordPress Multilingual Plugin). The vulnerability allowed attackers to execute arbitrary SQL queries, leading to potential data theft or manipulation. This vulnerability was assigned the CVE identifier CVE-2021-27902.

#### Impact of CVE-2021-27902

The vulnerability affected versions of WPML prior to 4.4.8. Attackers could exploit this flaw to gain unauthorized access to sensitive information stored in the database, such as user credentials and personal data. This highlights the importance of keeping plugins and software up-to-date to mitigate such risks.

### Out-of-Band SQL Injection

Out-of-band SQL Injection is a technique where the attacker triggers an interaction with an external system that they control. This method is particularly useful in scenarios where traditional SQL Injection techniques might not provide immediate feedback due to network restrictions or other constraints.

#### How Out-of-Band SQL Injection Works

In an out-of-band SQL Injection attack, the attacker crafts a payload that causes the target system to perform an action that interacts with an external domain controlled by the attacker. This interaction can be a DNS lookup, an HTTP request, or any other network activity that the attacker can monitor.

For example, consider a scenario where an attacker wants to confirm whether a SQL Injection vulnerability exists. They can craft a payload that causes the target system to perform a DNS lookup on a domain controlled by the attacker. If the DNS lookup occurs, the attacker knows that their payload was executed successfully.

### Lab 15: Blind SQL Injection with Out-of-Band Interaction

In this lab, you will exploit a SQL Injection vulnerability to cause a DNS lookup to a Burp Collaborator server. Burp Collaborator is a tool provided by Burp Suite Professional that allows you to monitor interactions with external domains.
