## Introduction to JWT and Its Vulnerabilities

Welcome to the Web Security Academy series, where we delve deep into the intricacies of securing web applications. Today, we will explore a specific type of vulnerability related to JSON Web Tokens (JWTs), specifically focusing on the "JOT authentication bypass via algorithm confusion" lab. This lab is part of the Web Security Academy hosted on PortSwigger.net, which provides comprehensive training materials for web security professionals.

### What is JWT?

JSON Web Token (JWT) is a compact, URL-safe means of representing claims to be transferred between two parties. It allows you to encode information in a token that can be verified and trusted because it is digitally signed. JWTs can be signed using a secret (with the HMAC algorithm) or a public/private key pair using RSA or ECDSA.

#### Structure of a JWT

A JWT consists of three parts separated by dots (`.`):

1. **Header**: Contains metadata about the token, such as the type of token and the signing algorithm used.
2. **Payload**: Contains the claims, which are statements about an entity (typically the user) and additional data.
3. **Signature**: Used to verify the integrity of the token. It is generated by hashing the encoded header and payload with a secret or a private key.

Here is an example of a JWT:

```plaintext
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkpvaG4gRG9lIiwiaWF0IjoxNTE2MzEwMDIxLCJleHAiOjE1MTYzMTEwMjF9.5rQDvqk4PnJbHhLWVZs6QKXoJyJvZB4vT6j75j7j7j7
```

- **Header**:
  ```json
  {
    "alg": "HS256",
    "typ": "JWT"
  }
  ```

- **Payload**:
  ```json
  {
    "sub": "1234567890",
    "name": "John Doe",
    "iat": 1516239021,
    "exp": 1516240021
  }
  ```

- **Signature**:
  ```plaintext
  5rQDvqk4PnJbHhLWVZs6QKXoJyJvZB4vT6j75j7j7j7
  ```

### Why JWT Matters

JWTs are widely used in web applications for authentication and authorization purposes. They provide a stateless way to manage user sessions, which can be beneficial for distributed systems. However, JWTs also introduce potential security risks if not implemented correctly.

### How JWT Works Under the Hood

When a user logs in, the server generates a JWT and sends it back to the client. The client stores this token and includes it in subsequent requests to the server. The server verifies the signature of the token to ensure its authenticity and checks the claims to determine whether the user is authorized to perform certain actions.

### Common JWT Flaws

One of the most critical vulnerabilities associated with JWTs is the "algorithm confusion" attack. This occurs when the server does not properly validate the signing algorithm used in the token, allowing an attacker to bypass authentication by using a weaker or different algorithm.
