## How to Prevent / Defend Against CSRF

### Secure Coding Practices

To prevent CSRF attacks, web applications should implement robust mechanisms to verify the authenticity of requests. Here are some secure coding practices:

#### Anti-CSRF Tokens

An anti-CSRF token is a unique value generated by the server and included in both the form and the session. When the form is submitted, the server verifies that the token in the form matches the one in the session.

**Vulnerable Code:**

```html
<form method="POST" action="/change-email">
    <input type="hidden" name="email" value="attacker@example.com">
</form>
```

**Secure Code:**

```html
<form method="POST" action="/change-email">
    <input type="hidden" name="email" value="attacker@example.com">
    <input type="hidden" name="csrf_token" value="{{ csrf_token }}">
</form>
```

**Server-Side Verification:**

```python
def change_email(request):
    if request.method == 'POST':
        email = request.POST.get('email')
        csrf_token = request.POST.get('csrf_token')
        if csrf_token == request.session['csrf_token']:
            # Process the email change
            return HttpResponse("Email address successfully changed")
        else:
            return HttpResponseForbidden("Invalid CSRF token")
```

### Validate the `Referer` Header

Another approach is to validate the `Referer` header to ensure that the request is coming from the expected origin.

**Raw HTTP Request:**

```http
POST /change-email HTTP/1.1
Host: vulnerable-app.com
Referer: https://vulnerable-app.com/
Content-Type: application/x-www-form-urlencoded
Cookie: session=abc123

email=attacker%40example.com
```

**Server-Side Validation:**

```python
def change_email(request):
    if request.method == 'POST':
        email = request.POST.get('email')
        referer = request.META.get('HTTP_REFERER')
        if referer and referer.startswith('https://vulnerable-app.com/'):
            # Process the email change
            return HttpResponse("Email address successfully changed")
        else:
            return HttpResponseForbidden("Invalid Referer header")
```

### Double-Submit Cookies

Double-submit cookies involve setting a cookie with a random value and including that value in the form. The server then verifies that the value in the form matches the value in the cookie.

**Setting the Cookie:**

```javascript
document.cookie = "csrf_token=" + Math.random().toString(36).substr(2);
```

**Form Submission:**

```html
<form method="POST" action="/change-email">
    <input type="hidden" name="email" value="attacker@example.com">
    <input type="hidden" name="csrf_token" value="{{ csrf_token }}">
</form>
```

**Server-Side Verification:**

```python
def change_email(request):
    if request.method == 'POST':
        email = request.POST.get('email')
        csrf_token = request.POST.get('csrf_token')
        cookie_csrf_token = request.COOKIES.get('csrf_token')
        if csrf_token == cookie_csrf_token:
            # Process the email change
            return HttpResponse("Email address successfully changed")
        else:
            return HttpResponseForbidden("Invalid CSRF token")
```

### Detection and Mitigation

To detect and mitigate CSRF attacks, organizations should implement logging and monitoring mechanisms to track suspicious activities. Additionally, regular security audits and penetration testing can help identify and remediate vulnerabilities.
