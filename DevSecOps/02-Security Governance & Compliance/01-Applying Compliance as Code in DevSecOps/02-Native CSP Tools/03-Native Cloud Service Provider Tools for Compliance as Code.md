## Native Cloud Service Provider Tools for Compliance as Code

### AWS Config

AWS Config is a service provided by Amazon Web Services (AWS) that enables organizations to assess, audit, and record changes to their AWS resource configurations. It helps in maintaining compliance by providing a detailed view of the resources associated with your AWS account, including how they are configured, how they are related to one another, and how their configurations and relationships have changed over time.

#### Key Features of AWS Config

1. **Configuration Recording**: Tracks changes to AWS resources and records them in a central repository.
2. **Rules Engine**: Allows you to define custom rules to check for compliance against specific criteria.
3. **Compliance Reports**: Generates reports that provide insights into the compliance status of your AWS resources.
4. **Integration with Other AWS Services**: Seamlessly integrates with other AWS services like CloudTrail, CloudWatch, and SNS for enhanced monitoring and alerting.

#### Setting Up AWS Config

To set up AWS Config, follow these steps:

1. **Enable AWS Config**:
    - Navigate to the AWS Management Console.
    - Go to the AWS Config dashboard.
    - Click on "Get Started" to enable AWS Config.

2. **Configure Recording**: 
    - Select the resources you want to track.
    - Configure the recording frequency.

3. **Create Rules**: 
    - Define custom rules to check for compliance.
    - Use the AWS Config rule builder to create rules.

4. **Generate Reports**: 
    - Use the AWS Config console to generate compliance reports.

#### Example: AWS Config Rule for S3 Bucket Encryption

Here’s an example of an AWS Config rule that ensures all S3 buckets are encrypted:

```yaml
{
  "ConfigRuleName": "s3-bucket-encryption",
  "Description": "Checks that all S3 buckets are encrypted.",
  "Scope": {
    "ComplianceResourceTypes": [
      "AWS::S3::Bucket"
    ]
  },
  "Source": {
    "Owner": "AWS",
    "SourceIdentifier": "S3_BUCKET_ENCRYPTION_ENABLED"
  }
}
```

#### How to Prevent / Defend

**Detection**:
- Use AWS Config to monitor and report on non-compliant S3 buckets.
- Set up alerts via AWS SNS to notify administrators of compliance violations.

**Prevention**:
- Enforce encryption policies using AWS Config rules.
- Use AWS Organizations to apply consistent compliance policies across multiple accounts.

**Secure-Coding Fix**:
- Ensure all S3 buckets are encrypted by default.
- Use IAM policies to restrict access to unencrypted buckets.

```yaml
# Vulnerable Configuration
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": "s3:*",
      "Resource": "*"
    }
  ]
}

# Secure Configuration
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Deny",
      "Action": "s3:PutObject",
      "Resource": "arn:aws:s3:::*",
      "Condition": {
        "StringNotEquals": {
          "s3:x-amz-server-side-encryption": ["AES256", "aws:kms"]
        }
      }
    }
  ]
}
```

### AWS Compliance Manager

AWS Compliance Manager is a tool that helps organizations manage and demonstrate compliance with various regulatory standards. It provides a centralized view of compliance status and helps in tracking progress towards achieving compliance goals.

#### Key Features of AWS Compliance Manager

1. **Compliance Frameworks**: Supports multiple compliance frameworks such as PCI-DSS, HIPAA, GDPR, etc.
2. **Control Mapping**: Maps AWS services to compliance controls.
3. **Evidence Collection**: Collects evidence of compliance from AWS services.
4. **Reporting**: Generates compliance reports for audits and assessments.

#### Setting Up AWS Compliance Manager

To set up AWS Compliance Manager, follow these steps:

1. **Select Compliance Framework**: Choose the compliance framework relevant to your organization.
2. **Map Controls**: Map AWS services to compliance controls.
3. **Collect Evidence**: Use AWS Config and other services to collect evidence.
4. **Generate Reports**: Generate compliance reports for audits and assessments.

#### Example: Compliance Report for PCI-DSS

Here’s an example of a compliance report generated by AWS Compliance Manager for PCI-DSS:

```json
{
  "ComplianceFramework": "PCI-DSS",
  "Controls": [
    {
      "ControlID": "3.1",
      "Description": "Protect stored cardholder data",
      "Status": "Compliant",
      "Evidence": [
        {
          "Type": "AWS Config",
          "Details": {
            "ResourceType": "AWS::EC2::Instance",
            "ComplianceType": "EncryptionEnabled"
          }
        }
      ]
    }
  ]
}
```

#### How to Prevent / Defend

**Detection**:
- Use AWS Compliance Manager to monitor compliance status.
- Set up alerts for non-compliant controls.

**Prevention**:
- Apply consistent compliance policies across all AWS resources.
- Use AWS Config rules to enforce compliance.

**Secure-Coding Fix**:
- Ensure all sensitive data is encrypted.
- Use IAM policies to restrict access to sensitive data.

```json
# Vulnerable Configuration
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": "ec2:*",
      "Resource": "*"
    }
  ]
}

# Secure Configuration
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Deny",
      "Action": "ec2:RunInstances",
      "Resource": "arn:aws:ec2:*:*:instance/*",
      "Condition": {
        "StringNotEquals": {
          "ec2:EncryptionType": "AES256"
        }
      }
    }
  ]
}
```

### Open Source Tools for Compliance as Code

Open source tools are also available for compliance as code:

- **Cloud Custodian**: An open-source tool for managing and enforcing compliance policies across cloud resources.

#### Key Features of Cloud Custodian

1. **Policy Definition**: Allows you to define compliance policies in YAML.
2. **Resource Management**: Manages cloud resources based on defined policies.
3. **Monitoring and Alerting**: Monitors resources and sends alerts for compliance violations.

#### Setting Up Cloud Custodian

To set up Cloud Custodian, follow these steps:

1. **Install Cloud Custodian**: Install Cloud Custodian using pip.
2. **Define Policies**: Write compliance policies in YAML.
3. **Run Policies**: Run policies to enforce compliance.

#### Example: Cloud Custodian Policy for S3 Bucket Encryption

Here’s an example of a Cloud Custodian policy that ensures all S3 buckets are encrypted:

```yaml
policies:
  - name: s3-bucket-encryption
    resource: s3
    filters:
      - type: bucket-encryption
        key: SSEAlgorithm
        value: AES256
```

#### How to Prevent / Defend

**Detection**:
- Use Cloud Custodian to monitor and report on non-compliant S3 buckets.
- Set up alerts via email or webhook to notify administrators of compliance violations.

**Prevention**:
- Enforce encryption policies using Cloud Custodian.
- Use IAM policies to restrict access to unencrypted buckets.

### Conclusion

Compliance as Code is a critical practice in DevSecOps that helps organizations maintain compliance consistently and efficiently. By leveraging native cloud service provider tools, open source tools, and third-party commercial tools, organizations can automate the enforcement of compliance policies and ensure ongoing compliance. Regular monitoring and auditing are essential to detect and prevent compliance violations, ensuring the security and integrity of the organization’s resources.

By following the steps outlined in this chapter and using the provided examples and labs, you can effectively implement compliance as code in your DevSecOps practices.
