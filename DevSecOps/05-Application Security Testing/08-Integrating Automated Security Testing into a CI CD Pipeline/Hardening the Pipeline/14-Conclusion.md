## Conclusion

Hardening a CI/CD pipeline involves several key steps, including setting up firewall access control lists, securing the artifact repository, and keeping software and plugins updated. Each of these steps plays a critical role in maintaining the security and integrity of the pipeline. By following best practices and implementing robust security measures, you can significantly reduce the risk of security incidents and ensure that your pipeline remains secure and reliable.

### Practice Labs

For hands-on practice in integrating automated security testing into a CI/CD pipeline, consider the following labs:

- **PortSwigger Web Security Academy**: Offers comprehensive training on web application security, including CI/CD pipeline security.
- **OWASP Juice Shop**: A deliberately insecure web application for practicing security testing and hardening techniques.
- **CloudGoat**: Provides scenarios for practicing cloud security, including CI/CD pipeline hardening.

By engaging with these labs, you can gain practical experience in applying the concepts discussed in this chapter.
