## Introduction to AWS and Automated Security Testing

Welcome to the final module of the DevSecOps course, which focuses on integrating automated security testing into Amazon Web Services (AWS) pipelines. In this module, we will explore how to effectively incorporate security testing into your existing AWS infrastructure, ensuring that your applications are robust and secure throughout their development lifecycle.

### Scenario Overview

Let's revisit the scenario involving Maeve and Jennifer. Maeve is responsible for managing an AWS pipeline and wants to integrate automated security testing into her workflow. Jennifer, a seasoned consultant, is tasked with providing guidance on how to achieve this goal. This module will delve into the various approaches and techniques Jennifer can employ to integrate automated security testing into AWS pipelines.

### Module Objectives

By the end of this module, you will understand:

- The importance of integrating automated security testing into AWS pipelines.
- Different methods to integrate security testing into existing AWS pipelines.
- How to modify an existing AWS pipeline to include automated security testing.
- Best practices for securing your AWS environment through automated testing.

### Caveats

Before we proceed, it is important to note that this module does not cover:

- Setting up a new AWS pipeline from scratch.
- Using AWS Security Hub or Amazon CloudWatch for monitoring and alerting.
- Detailed setup instructions for these services.

If you need information on these topics, consider exploring additional resources such as the Pluralsight Library, which offers comprehensive courses on AWS Security Hub and CloudWatch.
