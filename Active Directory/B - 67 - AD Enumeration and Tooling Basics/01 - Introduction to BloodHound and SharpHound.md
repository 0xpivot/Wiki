# 67.01 Introduction to BloodHound and SharpHound

## 1. Overview and Core Philosophy

Active Directory (AD) environments are highly complex and typically evolve over years or decades. This evolution results in a maze of nested groups, complex Access Control Lists (ACLs), Group Policy Object (GPO) links, and session data. For attackers and defenders alike, understanding the full scope of privileges and relationships is practically impossible using traditional LDAP queries or visual AD inspection tools alone.

BloodHound is an open-source, single-page JavaScript web application built on top of Linkurious, compiled with Electron, with a Neo4j database fed by a C# data collector called SharpHound. BloodHound uses graph theory to reveal the hidden and often unintended relationships within an Active Directory or Azure environment. 

### 1.1 The Graph Theory Approach
Traditional enumeration produces lists (e.g., a list of Domain Admins). Graph theory connects these lists. In AD, graph theory represents data as:
- **Nodes (Vertices)**: Users, Groups, Computers, Domains, GPOs, Organizational Units (OUs).
- **Edges (Relationships)**: The permissions or states that connect nodes. Examples include `MemberOf`, `AdminTo`, `HasSession`, `GenericAll`, `WriteDacl`, `CanRDP`, etc.

By calculating the shortest paths between a starting node (e.g., a compromised low-privileged user) and a target node (e.g., the Domain Admins group), BloodHound uncovers complex attack paths that would take humans weeks to deduce manually.

## 2. Architecture and Data Flow

The BloodHound ecosystem consists of three distinct components working together: the ingestor, the database, and the visualization UI.

```text

|                       |       |                        |       |                        |
```

### 2.1 SharpHound (The Collector)
SharpHound is the tool that actively queries the target environment. It is typically executed from a domain-joined machine or via compromised credentials. It communicates with Domain Controllers using LDAP to pull directory objects and permissions. It also communicates with individual endpoints via RPC and SMB to enumerate active user sessions and local administrator groups.

### 2.2 Neo4j (The Database)
Neo4j is a highly scalable, native graph database. It stores the JSON output generated by SharpHound. Neo4j is queried using the Cypher Query Language, which is specifically designed for traversing graphs.

### 2.3 BloodHound UI (The Visualizer)
The Electron application where the operator interacts with the data, runs queries, and visualizes the attack paths graphically.

## 3. SharpHound Data Collection Mechanisms

SharpHound relies on several distinct collection methods, each with its own network footprint and prerequisite permissions.

### 3.1 LDAP Queries (The Foundation)
Most of BloodHound's data comes from LDAP queries directed at Domain Controllers.
- **Mechanism**: SharpHound connects to TCP port 389 (or 636 for LDAPS) on the DC. It performs extensive searches for users, groups, computers, OUs, and GPOs. Crucially, it requests the `nTSecurityDescriptor` attribute for objects to map out complex ACLs.
- **Requirements**: Any authenticated domain user (and sometimes even a computer account or guest account) can read this information by default.
- **Data Yield**: `MemberOf`, `GenericAll`, `WriteDacl`, `Owns`, etc.

### 3.2 Local Group Enumeration (Finding the Admins)
Understanding who is a local administrator on which machine is critical for lateral movement.
- **Mechanism**: SharpHound queries the Security Account Manager (SAM) via the SAMR protocol or the `NetLocalGroupGetMembers` API over RPC (TCP 445/135) against every computer object discovered via LDAP.
- **Requirements**: Historically, any domain user could query this. However, Windows 10 Anniversary Update and later restrict SAMR querying to local administrators by default via the `Network access: Restrict clients allowed to make remote calls to SAM` policy.
- **Data Yield**: `AdminTo`, `CanRDP`, `ExecuteDCOM`.

### 3.3 Session Enumeration (The Volatile Links)
Session data determines where users are currently logged in. This is the most dynamic and critical data for finding immediate attack paths.
- **Mechanism**: SharpHound queries the `NetSessionEnum` API on target machines via SMB/RPC.
- **Requirements**: By default, this API requires local administrator privileges on the target endpoint. However, Domain Controllers and File Servers often allow unprivileged users to query session data due to legacy configurations.
- **Data Yield**: `HasSession`.

## 4. Execution and Usage

### 4.1 SharpHound Syntax and Options
Running SharpHound effectively requires understanding its flags to balance data collection needs with OPSEC considerations.

```powershell
# Basic collection (Default: Group, LocalAdmin, Session, Logging, Trust, SPNTargets)
Invoke-BloodHound -CollectionMethod Default

# All collection methods (includes volatile session data and all ACLs - VERY NOISY)
Invoke-BloodHound -CollectionMethod All

# Targeting a specific domain
Invoke-BloodHound -CollectionMethod Default -Domain target.local

# Stealthy collection (Randomizes delays, limits threads, skips aggressive checks)
Invoke-BloodHound -CollectionMethod Stealth

# Looping collection to catch volatile session data over time
Invoke-BloodHound -CollectionMethod Session -Loop -LoopDuration 02:00:00
```

#### Key Parameters:
- `-CollectionMethod`: Dictates what data to pull. Options include `Default`, `Group`, `LocalGroup`, `RDP`, `DCOM`, `PSRemote`, `Session`, `ObjectProps`, `ACL`, `ComputerOnly`, `All`.
- `-Domain`: Specify the target domain if different from the current context.
- `-SearchForest`: Expands the search to all domains within the forest.
- `-Stealth`: Limits concurrent connections and introduces jitter to evade basic detection.
- `-ExcludeDCs`: Useful if DCs are heavily monitored for API calls.

### 4.2 Ingesting Data
Once SharpHound completes, it outputs a compressed ZIP file containing multiple JSON files (e.g., `users.json`, `computers.json`, `gpos.json`).
1. Start the Neo4j console service.
2. Launch the BloodHound UI and authenticate.
3. Drag and drop the ZIP file into the UI or use the upload button.
4. Wait for the database to process and map the relationships.

## 5. Analyzing Data and Cypher Queries

### 5.1 Built-in Analytics
BloodHound provides pre-built queries in the "Analysis" tab that cover 90% of standard use cases:
- **Find all Domain Admins**: Identifies the primary targets.
- **Find Shortest Paths to Domain Admins**: The most critical query; shows how the current user can elevate privileges.
- **Find Principals with DCSync Rights**: Identifies objects that can replicate AD secrets (e.g., NTDS.dit) without needing to be a Domain Admin.
- **Map Domain Trusts**: Visualizes trust relationships, highlighting potential cross-domain attacks.
- **Shortest Paths from Kerberoastable Users**: Combines credential extraction techniques with lateral movement paths.

### 5.2 Advanced Cypher Queries
For complex analysis, operators must write raw Cypher queries.

**Example 1: Find all Kerberoastable users with an administrative path**
```cypher
MATCH (u:User {hasspn:true})
MATCH (g:Group {name:'DOMAIN ADMINS@DOMAIN.LOCAL'})
MATCH p = shortestPath((u)-[*1..]->(g))
RETURN p
```
*Explanation: Finds users with SPNs (which can be cracked offline) that also have a direct or indirect path to Domain Admins.*

**Example 2: Find all computers where a specific user is a local admin**
```cypher
MATCH (u:User {name:"COMPROMISED_USER@DOMAIN.LOCAL"})-[r:AdminTo]->(c:Computer) 
RETURN u,r,c
```

**Example 3: Find paths using only ACL abuses (ignoring active sessions)**
```cypher
MATCH p=shortestPath((u:User)-[r:GenericAll|GenericWrite|WriteOwner|WriteDacl|ForceChangePassword*1..]->(t:Group {objectid: "S-1-5-21-XXX-512"})) 
RETURN p
```

## 6. Edge Definitions and Abuse Primitives

Understanding what an edge means is critical for executing the attack path.
- **GenericAll**: Full control over the target object. The attacker can reset passwords, modify group membership, or change SPNs.
- **GenericWrite**: Ability to modify attributes of the target. Attackers can write to `msDS-AllowedToActOnBehalfOfOtherIdentity` to perform resource-based constrained delegation (RBCD).
- **WriteDacl**: Ability to modify the ACL of the target. The attacker can grant themselves `GenericAll`.
- **ForceChangePassword**: Ability to reset a user's password without knowing their current password. Highly disruptive but effective.
- **AddMembers**: Ability to add any user (including the attacker) to a target security group.
- **AdminTo**: The attacker has local administrative rights on the target computer, allowing them to dump LSASS, execute code via WMI/PsExec, or install malware.

## 7. OPSEC and Detection Strategies

### 7.1 The Noise of SharpHound
SharpHound is inherently noisy.
- **Network Traffic**: It generates massive amounts of LDAP, RPC, and SMB traffic. Sweeping a /16 network for sessions will trigger internal port scan alerts.
- **Log Generation**: Querying thousands of machines for local groups and sessions generates countless Event ID 4624 (Logon) and 4679 (Special Privileges) events if System Access Control Lists (SACLs) are configured.
- **Signatures**: `SharpHound.exe` and its PowerShell variants are heavily signatured by all modern Antivirus (AV) and Endpoint Detection and Response (EDR) solutions.

### 7.2 Detection for Defenders
1. **LDAP Anomaly Detection**: Monitor for high volumes of LDAP queries requesting `nTSecurityDescriptor` from non-Domain Controller IPs.
2. **SMB/RPC Sweeps**: Alert on endpoints establishing IPC$ or RPC connections (port 445/135) to hundreds of distinct internal IPs within a short timeframe.
3. **Process Execution**: Hunt for known SharpHound arguments in process command lines via Event ID 4688 (e.g., `-CollectionMethod All`, `--stealth`).
4. **Honey Tokens**: Create a highly privileged fake account (e.g., `svc_domain_backup`) with complex ACLs and monitor for any user querying its properties in AD.

## 8. Mitigation and Remediation

Defenders should use BloodHound offensively against their own networks to identify and close attack paths.
- **Implement AD Tiering**: Use Microsoft's Tiered Administration Model. Prevent Tier 0 administrators (Domain Admins) from logging into Tier 2 workstations. This completely breaks `HasSession` edges that allow lateral escalation.
- **Restrict SAMR**: Deploy Group Policy to disable remote SAM querying, which prevents attackers from easily mapping `AdminTo` edges.
- **Clean up Legacy ACLs**: Identify and remove unintended permissions, such as nested groups granting `GenericAll` to standard users over critical infrastructure.
- **Enforce LAPS**: Use the Local Administrator Password Solution to randomize local admin passwords, preventing lateral movement via Pass-the-Hash.

## Real-World Attack Scenario

An attacker secures low-privileged domain credentials and executes SharpHound from a compromised workstation, using stealth mode (`--stealth`) to bypass simple threshold-based LDAP alerts. The data is exfiltrated and ingested into the attacker's offline BloodHound database. Through Cypher queries, the attacker uncovers an unintended privilege chain: their low-privileged user has `GenericWrite` permissions over a Tier 2 support group, which in turn has `AdminTo` rights on a sensitive file server. By leveraging this previously invisible graph relationship, the attacker adds themselves to the support group, moves laterally to the file server, and continues their path toward full domain compromise.

## Advanced Threat Hunting and Behavioral Analytics

As evasion techniques evolve, reliance on static indicators of compromise (IoCs) is insufficient. Defenders must pivot to behavioral analytics.

### Baseline Deviation Analysis
Instead of hunting for specific tool signatures (like `SharpHound.exe` or `PowerView.ps1`), mature SOCs establish baselines of administrative behavior.
1. **Administrative Logon Baselines**: Identify the standard jump boxes and IP ranges used by authorized administrators. Any high-privileged authentication originating from a non-standard workstation (e.g., a receptionist's PC) triggers an immediate severity-high alert, regardless of the tool used.
2. **Protocol Baselines**: Standard users rarely, if ever, initiate raw RPC or WMI connections to other workstations. Detecting a high volume of lateral SMB/RPC traffic originating from a standard subnet is a strong behavioral indicator of enumeration or lateral movement.

### Leveraging Graph Databases for Defense
Defenders can utilize the exact same graph theory concepts employed by attackers to secure the environment proactively.
- **Continuous Ingestion**: By scheduling daily or weekly automated ingestions of AD data into a defensive Neo4j database, defenders can track changes over time.
- **Chokepoint Identification**: Graph analysis reveals "chokepoints"—specific users or groups that serve as critical bridges in numerous attack paths. Removing privileges from these chokepoint accounts fractures the attack graph, significantly increasing the effort required by an attacker.
- **Unintended Permission Auditing**: Graph databases easily highlight misconfigurations such as standard users accidentally granted `GenericAll` rights over critical infrastructure OUs due to complex, nested group memberships.

### Conclusion
Active Directory enumeration is a delicate balance of noise versus insight. Attackers must constantly adapt to increasingly sophisticated telemetry and detection mechanisms. For defenders, understanding the mechanics of these enumeration tools is paramount. Security is no longer just about preventing the initial compromise; it is about anticipating the attacker's post-exploitation reconnaissance and disrupting their ability to discover the pathways to domain dominance.

### Real-World Incident Response Scenarios
When responding to suspected AD enumeration:
1. **Isolate Suspect Endpoints**: Immediately quarantine the endpoint initiating the anomalous LDAP or RPC queries.
2. **Review TGT Requests**: Correlate the endpoint activity with Event ID 4768 (A Kerberos authentication ticket (TGT) was requested) to identify the compromised account.
3. **Analyze BloodHound Execution**: If SharpHound is suspected, search for Event ID 4688 containing command-line arguments like `--CollectionMethod` or `--Loop`.
4. **Implement Tiered Administration**: If widespread enumeration is detected, immediately restrict Tier-0 accounts from authenticating to lower-tier systems to break potential `HasSession` escalation paths.

## Glossary of Advanced Terms
- **TGT (Ticket Granting Ticket)**: The primary Kerberos ticket used to request access to other services.
- **SPN (Service Principal Name)**: A unique identifier for a service instance, used in Kerberos authentication.
- **ACL (Access Control List)**: A list of permissions attached to an object.
- **DCSync**: An attack technique simulating a Domain Controller to request password hashes via the Directory Replication Service (DRS) Remote Protocol.
- **Pass-the-Hash (PtH)**: Authenticating to a remote system using the underlying NTLM hash of a user's password, rather than the plaintext password itself.
